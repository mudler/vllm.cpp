## Scope

Add `MistralForCausalLM` (Mistral-7B-v0.3: standard dense transformer — GQA 32/8, head_dim 128, SwiGLU, standard non-gemma RMSNorm, **untied lm_head**, `sliding_window: null` → one full-attention KV group) as the fifth model family. The two Llama deltas both resolve to REUSE, with **NO new primitive**: (1) **NO per-head q/k RMSNorm** (like Llama) → the shared `dense_attn_block.h::AttnBlock` skips the norm step when q_norm/k_norm are empty (`has_qk_norm`, landed for Llama); (2) **plain RoPE, theta=1000000, NO rope_scaling** → `rope_type=="default"`, so `MakeRopeArgs(cfg)` leaves the llama3 fields zero and the RoPE kernels take their no-op branch — the exact plain-rope path Qwen3-dense uses, ZERO new kernel (SIMPLER than Llama-3.2's llama3 rescale). Two config-shape differences vs Llama-3.2-1B, both already handled generically: **UNTIED embeddings** (`tie_word_embeddings=false` → a standalone `lm_head` is loaded, the untied branch the shared loader/forward already have) and a **`sliding_window` field present but null** (SWA disabled in v0.3 → full attention). In scope: config consumption (already parsed by `LoadHfConfig`), registry TU, full-attention-only KV spec, weight loader, the SACRED near-tie-robust token-exact gate. Out of scope: SWA-enabled Mistral variants (v0.1, window 4096 — `SlidingWindowSpec` has no production construction site here, so such a variant would still use full attention; documented, separate row), `llama_4_scaling` (Llama-4-specific attention scaling in `MistralAttention`, absent in Mistral-7B), Ministral/Mistral-Large-3, NVFP4/GGUF Mistral, SPEED close (a separate increment).
