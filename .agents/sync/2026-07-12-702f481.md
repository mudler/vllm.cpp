## Retirement and modernization decision

1. **Do not delete local paged-KV attention.** Upstream removed only
   `csrc/libtorch_stable/attention/{attention_kernels.cuh,paged_attention_v1.cu,paged_attention_v2.cu}`.
   The local tree has no reference to those files or symbols. Its
   `vt::PagedAttention` is the backend-neutral paged-KV contract used by the
   current FA2 and fallback implementations, so deleting it would remove live
   functionality.
2. **Do not delete the removed-model names.** Baichuan, Aquila, Grok, Tarsier,
   AyaVision, MusicFlamingo and Mantis exist locally only in the pinned
   previously-supported rejection table. Retaining those aliases preserves
   vLLM-compatible diagnostics; they are not executable model implementations.
3. **Keep the FP4 topology repair, but rebase its evidence.** v0.25.0 still
   calls NVFP4 quantization with `is_sf_swizzled_layout=True`, still
   zero-initializes padding the producer does not write, and still supplies a
   model-owned device `alpha` pointer to CUTLASS. Our trace-grounded
   direct-swizzled-scale and resident-alpha work is therefore current, but no
   implementation begins until a fresh v0.25.0 trace confirms the gap.
4. **Evaluate blocking CUDA events by trace, not assumption.** vllm#47081 is a
   new denominator-side performance change. Mirror it only if the new paired
   trace shows an equivalent local busy-wait/driver-lock cost.
5. **Correct the oracle dependency drift first.** The retired DGX oracle was
   vLLM 0.24.0 + FlashInfer 0.6.12 even though the current source pin already
   requires FlashInfer 0.6.13. All old-oracle rates are diagnostic only. The
   replacement v0.25.0 environment now fingerprints vLLM, FlashInfer, Torch,
   CUTLASS DSL, Humming, Ninja and the CLI; lock-held real 27B production-graph
   and server smoke pass, the canonical path is active, and v0.24 rollback is
   preserved. No denominator binds until the fresh exact grid reproduces.
