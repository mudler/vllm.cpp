# M0.1 Build Skeleton Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A building, tested, CI-green `libvllm` skeleton: CMake (host + optional CUDA sm_121), vendored header-only deps, doctest-based test harness, GitHub Actions CI, and the layout conventions every later milestone inherits.

**Architecture:** Single static library target `vllm` (public headers under `include/`, sources mirroring upstream vLLM paths under `src/vllm/`, compute runtime later under `src/vt/`). CUDA is an optional, auto-detected component gated by `VLLM_CPP_CUDA` — CI builds CPU-only; `dgx.casa` builds with sm_121. Tests are doctest executables registered with CTest.

**Tech Stack:** C++20, CMake ≥ 3.24, doctest (vendored), nlohmann/json + cpp-httplib (vendored for M0.4/M3), CUDA 13.x toolchain on dgx.casa only, GitHub Actions (ubuntu-latest).

## Global Constraints

- No PyTorch, no ggml — no compiled third-party deps at all in M0.1; vendored single-header libs only (`.agents/discipline.md`).
- Mirrored-structure rule applies from the first file: engine sources will live at `src/vllm/<upstream-path>`; nothing in this milestone may contradict that layout (design doc §4).
- Warnings are errors for our code (`-Wall -Wextra -Werror`); vendored headers are `SYSTEM` includes and exempt.
- CUDA architecture: `121` (GB10). CUDA must remain strictly optional — configuring without a CUDA toolchain must succeed silently.
- Push directly to `main`; every commit message plain-imperative per repo history.
- Close-the-loop DoD (`.agents/workflow.md`): roadmap status, parity-ledger row, state-log entry are part of this milestone's final task.

- [ ] **Step 1: Write `cmake/CompilerWarnings.cmake`**

```cmake
function(vllm_cpp_set_warnings target)
  target_compile_options(${target} PRIVATE
    $<$<COMPILE_LANGUAGE:CXX>:-Wall -Wextra -Werror>
    $<$<COMPILE_LANGUAGE:CUDA>:-Werror=all-warnings>)
endfunction()
```

- [ ] **Step 2: Write `CMakeLists.txt`**

```cmake
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_EXPORT_COMPILE_COMMANDS ON)
set(CMAKE_POSITION_INDEPENDENT_CODE ON)

option(VLLM_CPP_BUILD_TESTS "Build tests" ON)
option(VLLM_CPP_CUDA "Build the CUDA backend (auto-detected if unset)" OFF)
set(VLLM_CPP_CUDA_ARCHITECTURES "121" CACHE STRING "Target CUDA architectures (GB10=121)")

include(CheckLanguage)
if(NOT VLLM_CPP_CUDA)
  check_language(CUDA)
  if(CMAKE_CUDA_COMPILER)
    message(STATUS "CUDA toolchain found; enabling VLLM_CPP_CUDA")
    set(VLLM_CPP_CUDA ON)
  endif()
endif()
if(VLLM_CPP_CUDA)
  enable_language(CUDA)
  set(CMAKE_CUDA_STANDARD 20)
  set(CMAKE_CUDA_ARCHITECTURES ${VLLM_CPP_CUDA_ARCHITECTURES})
endif()

include(cmake/CompilerWarnings.cmake)

configure_file(include/vllm/version.h.in include/vllm/version.h) # written in Step 3

  src/vllm/version.cpp)
add_library(vllm::vllm ALIAS vllm)
target_include_directories(vllm
  PUBLIC include ${CMAKE_CURRENT_BINARY_DIR}/include)
target_include_directories(vllm SYSTEM PUBLIC third_party)
vllm_cpp_set_warnings(vllm)
if(VLLM_CPP_CUDA)
```

- [ ] **Step 2: Write `src/vt/cuda/smoke.cu`**

```cpp
#include <cuda_runtime.h>

#include <stdexcept>
#include <string>

#include "vt/cuda_smoke.h"

namespace vt::cuda {
namespace {
__global__ void AddKernel(int a, int b, int* out) { *out = a + b; }

void Check(cudaError_t err, const char* what) {
  if (err != cudaSuccess) {
    throw std::runtime_error(std::string(what) + ": " + cudaGetErrorString(err));
  }
}
}  // namespace

int SmokeAdd(int a, int b) {
  int* d_out = nullptr;
  Check(cudaMalloc(&d_out, sizeof(int)), "cudaMalloc");
  AddKernel<<<1, 1>>>(a, b, d_out);
  Check(cudaGetLastError(), "launch");
  int out = 0;
  Check(cudaMemcpy(&out, d_out, sizeof(int), cudaMemcpyDeviceToHost), "cudaMemcpy");
  Check(cudaFree(d_out), "cudaFree");
  return out;
}
}  // namespace vt::cuda
```
