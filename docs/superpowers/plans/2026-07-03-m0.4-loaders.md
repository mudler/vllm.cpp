# M0.4 Loaders Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Read the gate checkpoints from disk: HF `config.json` parsing, safetensors mmap reader, and a GGUF container reader (standard quants + the NVFP4 extension types from the killgate tooling) — with tensor bytes verified against Python readers on `dgx.casa`.

**Architecture:** Container readers expose *raw tensor bytes + metadata* (name → dtype/shape/data span); they do **not** decode quantized blocks (that's kernels' job, M0.6+/M2.2). Mirrored placement: `src/vllm/model_executor/model_loader/{safetensors_reader,gguf_reader}.{h,cpp}` (semantics ported from upstream `weight_utils.py`/`gguf_loader.py` where applicable) and `src/vllm/transformers_utils/hf_config.{h,cpp}` for config.json. mmap-based, read-only, loud `VT_CHECK`-style validation — file metadata is untrusted input.

**Tech Stack:** POSIX mmap, nlohmann/json (vendored), doctest. Python (dgx oracle venv) only for e2e byte-verification.

## Global Constraints

- File metadata is UNTRUSTED: every size/offset/count is bounds-checked against the actual file size before use; integer overflow paths guarded (this task also lands the carried-over `Tensor::Numel`/`Contiguous` overflow guards).
- Readers return views into the mmap (zero-copy); no quant decoding in this milestone.
- Upstream loyalty: GGUF name→param mapping conventions come later (M0.9); this milestone is containers only. File headers: `// Ported from: vllm/model_executor/model_loader/gguf_loader.py (container semantics) @ e24d1b24` for gguf_reader, `// vllm.cpp original (container reader); no upstream mirror.` for safetensors_reader (upstream defers to the Rust safetensors crate) and hf_config.
- Warnings-as-errors; TDD with synthetic in-test container bytes (no Python in CI); e2e vs real checkpoints on dgx.casa is the DoD gate.
- Push to `main` authorized. dgx SSH needs `export PATH=/usr/local/cuda/bin:$PATH` (not needed for these CPU steps but recorded).

### Task 1: Tensor overflow hardening (carryover)

**Files:** Modify `src/vt/tensor.cpp` (+ tests in `tests/vt/test_tensor.cpp`)

- `Tensor::Contiguous`: guard the stride accumulation — before `acc *= t.shape[d]`, `VT_CHECK(t.shape[d] == 0 || acc <= INT64_MAX / t.shape[d], "tensor shape overflow")` (shape dims are already checked positive).
- `Numel()`: same guard pattern on `n *= shape[d]`.
- Tests: `Tensor::Contiguous(buf, kF32, cpu, {1<<20, 1<<20, 1<<20, 2})` → throws; `Numel` on a hand-built Tensor with huge dims → throws.
- Commit `fix(vt): overflow guards in Tensor shape math`, push.

### Task 3: safetensors reader

**Files:** Create `include/vllm/model_executor/model_loader/safetensors_reader.h`, `src/vllm/model_executor/model_loader/safetensors_reader.cpp`; test `tests/vllm/test_safetensors.cpp`.

**Interfaces:**
```cpp
namespace vllm {
struct StTensor { std::string dtype;            // "F32","BF16","F16","U8","F8_E4M3","I64",...
                  std::vector<int64_t> shape;
                  const uint8_t* data; size_t nbytes; };
class SafetensorsFile {  // one .safetensors file, mmap'd read-only
 public:
  static SafetensorsFile Open(const std::string& path);   // throws on any malformation
  const std::vector<std::string>& Names() const;
  const StTensor& Get(const std::string& name) const;     // throws if absent
  const std::map<std::string, std::string>& Metadata() const;  // "__metadata__"
  // move-only; munmaps in dtor
};
// Multi-file checkpoints: parse model.safetensors.index.json → {tensor→file}.
std::map<std::string, std::string> LoadSafetensorsIndex(const std::string& index_json_path);
}
```
Format: `u64 LE header_len` + JSON header + data section. Validation (untrusted): header_len ≤ file_size-8; every `data_offsets [b,e]` satisfies b ≤ e ≤ data_section_size; nbytes == dtype_size*numel for known dtypes (unknown dtypes allowed — raw span kept, size not cross-checked but still bounds-checked); duplicate names throw.

Tests build a tiny valid file in-code (2 tensors bf16/f32 + metadata), write to /tmp, Open, verify names/shapes/bytes; malformed cases: truncated header, offsets beyond file, overlapping offsets allowed? (safetensors spec says non-overlapping ascending — enforce ascending non-overlap, test violation throws), huge header_len → throw. Commit `feat(loader): safetensors mmap reader`.

### Task 4: GGUF reader (standard types)

**Files:** Create `include/vllm/model_executor/model_loader/gguf_reader.h`, `src/vllm/model_executor/model_loader/gguf_reader.cpp`; test `tests/vllm/test_gguf.cpp`.

**Interfaces:**
```cpp
namespace vllm {
struct GgufValue { /* tagged union: u8..u64,i8..i64,f32,f64,bool,string,array (std::variant-based) */ };
struct GgufTensorInfo { std::string name; std::vector<int64_t> shape;  // ggml dims reversed to row-major torch order
                        uint32_t ggml_type; const uint8_t* data; size_t nbytes; };
class GgufFile {
 public:
  static GgufFile Open(const std::string& path);           // v3 (+v2 length-compat), LE only
  const GgufValue* FindKv(const std::string& key) const;   // nullptr if absent
  const std::vector<GgufTensorInfo>& Tensors() const;
  const GgufTensorInfo& Get(const std::string& name) const;
  // move-only; munmap in dtor
};
// type traits for size math: {block_elems, block_bytes} per ggml type
struct GgmlTypeTraits { int64_t block_elems; int64_t block_bytes; const char* name; };
const GgmlTypeTraits& GgmlTraits(uint32_t type);            // throws on unknown
}
```
Standard traits table (from ggml, recorded here): F32{1,4} F16{1,2} BF16{1,2}(type 30) Q8_0{32,34} Q4_0{32,18} Q4_K{256,144} Q5_K{256,176} Q6_K{256,210} Q3_K{256,110} Q2_K{256,84} I8/I16/I32/I64. `nbytes = numel/block_elems*block_bytes` (VT_CHECK numel % block_elems == 0). Alignment from `general.alignment` kv (default 32). All counts/offsets untrusted-validated. kv value types incl. arrays (needed for tokenizer vocab in M0.5 — string arrays must work).

Tests: synthetic GGUF built byte-by-byte in-test (header v3, 3 kvs incl. string + u32 + string-array, 2 tensors F32 and Q8_0), verify parse; malformed: bad magic, kv count huge → throw before allocation (cap sanity: kv count and tensor count ≤ 1e6), tensor offset beyond data section, dims overflow (reuse Task 1 guards via VT_CHECK on multiplication). Commit `feat(loader): GGUF v3 container reader (standard ggml types)`.
