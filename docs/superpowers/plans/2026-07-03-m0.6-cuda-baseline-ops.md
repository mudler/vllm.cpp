# M0.6 CUDA Baseline Ops Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A real CUDA backend for vt on the GB10 (sm_121): cuBLASLt GEMM + correctness-grade CUDA kernels for rmsnorm/silu_and_mul/embedding/rope, passing the SAME committed parity goldens as the CPU reference.

**Architecture:** `vt::cuda::CudaBackend` (registers `DeviceType::kCUDA`, replaces the M0.1 smoke placeholder) + kernels in `src/vt/cuda/` registering into the existing op tables. Ops accept bf16/f32 in and — after this milestone's validation widening — bf16 OR f32 out (goldens still compare via f32). Development and testing happen on dgx.casa over SSH; CI remains CPU-only (CUDA sources compile-gated as before). Parity: `test_op_parity` grows an optional CUDA pass (same 9 golden cases, run when a CUDA backend is registered), executed on the DGX.

**Tech Stack:** CUDA 13.0 (nvcc on dgx, sm_121), cuBLASLt (part of the toolkit; link `CUDA::cublasLt` — first non-header dependency, allowed: system toolkit component like cudart), doctest.

## Global Constraints

- Correctness-grade first: plain kernels, no fusion/tuning (that's M2). Bit-parity is NOT expected vs CPU scalar — the committed golden tolerances are the bar.
- Every carried M0.2 item lands here: kernel fn-type aliases move to `include/vt/ops.h` with `static_cast` at registration (compile-time signature tie); op validation widened to allow `out.dtype ∈ {kF32, kBF16}` (kernels may not support all combos — a kernel that doesn't must throw loudly, not silently truncate); Embedding's bounds-check contract documented for GPU (device-side ids cannot throw → CUDA embedding pre-validates ids on host? NO — ids live on device; decision: CUDA kernel clamps + records an error flag in a pinned int checked after sync, fail-loud on host).
- dgx.casa workflow: `export PATH=/usr/local/cuda/bin:$PATH`; build `cmake -S . -B build-cuda -DVLLM_CPP_CUDA=ON`; keep artifacts in `~/work/vllm.cpp`. Push-pull via GitHub as established.
- Streams: every kernel launches on the `Queue`'s `cudaStream_t` (`Queue::handle`); no default-stream usage; ops are async — tests sync via a new `Backend::Synchronize(Queue&)` virtual (CPU: no-op).
- Memory for tests: CUDA backend `Alloc` = `cudaMallocAsync`? No — keep simple `cudaMalloc` (M0 correctness); `Copy` handles h2d/d2h/d2d via `cudaMemcpyAsync(..., stream)` + the Backend gains `CopyKind` inference from pointer attributes? NO — keep the existing Copy signature; CUDA impl uses `cudaMemcpyDefault` (unified addressing resolves direction).
- Warnings-as-errors incl. CUDA (`-Werror=all-warnings`); vt header comments; TDD where the test can run (CPU-side validation tests locally; kernel tests are dgx-run).
- Push to `main` authorized.
