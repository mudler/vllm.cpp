> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Correctness-grade Gated-DeltaNet ops for the Qwen3.6 hybrid layers — causal conv1d (fwd + single-step update), L2 norm, gated RMSNorm, and the gated-delta-rule state recurrence (prefill over a sequence + decode single-step) — CPU and CUDA, passing layer-op goldens dumped from the **pinned** upstream checkout.

**Architecture:** Two deliberate scope decisions. (1) **Sequential over chunked:** upstream's `chunk_gated_delta_rule` is a parallelization (WY representation) of a sequential recurrence; for correctness-grade M0.7 we implement the recurrence directly (parallel over batch×head, sequential over tokens) and validate against the pinned chunked reference — M2.3 replaces it with a real chunked kernel for prefill throughput. (2) **Ops, not the module:** the QwenGatedDeltaNetAttention orchestration (projections, splits) is M0.9 model-assembly; M0.7 delivers the ops it calls. Workspace policy per M0.6 review: ops take explicit caller-allocated tensors; no hidden allocs, no forced syncs in hot paths.

**Tech Stack:** pinned checkout `/home/mudler/_git/vllm` @ e24d1b24 as the ONLY oracle (exec'd on dgx with the venv's torch+triton, PYTHONPATH/exec-stub techniques as in oracle_detokenizer.py); vt op registry; CUDA sm_121.
