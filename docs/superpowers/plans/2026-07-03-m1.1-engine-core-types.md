# M1.1 Engine Core Types — Request / SamplingParams / Outputs

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** The foundational data structures the V1 engine passes around — `SamplingParams`, `Request`/`EngineCoreRequest`, `RequestStatus`, `SamplerOutput`/`ModelRunnerOutput`, `EngineCoreOutput(s)`, `RequestOutput` — ported 1:1 from vLLM V1 so the scheduler (M1.4), model runner (M1.5), and EngineCore (M1.8) have their shared vocabulary. Behavioral (no model, no CUDA); pure C++ + unit tests in CI.

**Architecture:** Mirrored placement under `src/vllm/`: `sampling_params.{h,cpp}` (← `vllm/sampling_params.py`), `v1/request.{h,cpp}` (← `vllm/v1/request.py`), `v1/engine/types.{h,cpp}` (← `vllm/v1/engine/__init__.py` EngineCoreRequest/Output(s), `vllm/v1/outputs.py` ModelRunnerOutput/SamplerOutput), `outputs.{h,cpp}` (← `vllm/outputs.py` RequestOutput/CompletionOutput). Value types with clear ownership; the scheduler/runner mutate `Request` in place exactly as upstream does. This unit is DATA + small helpers only — no scheduling/execution logic.

**Tech Stack:** C++20, doctest. No CUDA, no model. Everything runs in CI.

## Global Constraints

- 1:1 with upstream V1: same field names (C++-case-adjusted only where unavoidable), same defaults, same enum values, same semantics. Every ported file carries `// Ported from: vllm/<path> @ e24d1b24`. Read the upstream BEFORE writing each struct.
- Scope = the T0 subset (from porting-inventory §3 sampling): the fields the M1 engine actually needs. Defer T1/T2 fields (structured outputs, logit_bias, bad_words, beam search, spec-decode fields, LoRA, multimodal, prompt_embeds) — but leave clearly-marked extension points so they slot in later without reshaping the struct.
- Validation mirrors upstream `SamplingParams.__post_init__` / `_verify_args` exactly (temperature ≥ 0, top_p ∈ (0,1], top_k semantics, penalties ranges, n ≥ 1, max_tokens, min_tokens ≤ max_tokens, stop vs stop_token_ids, seed) — throw on invalid (upstream raises ValueError).
- Warnings-as-errors; TDD; push authorized (CPU-only, all CI).
