# M1.4 Scheduler (unified token-budget)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** The V1 Scheduler — the unified token-budget algorithm (no prefill/decode distinction; each request tracks `num_computed_tokens` vs `num_tokens`, tokens assigned per step under a budget), chunked prefill, FCFS preemption (via `KVCacheManager.allocate_slots → nullopt`), and `SchedulerOutput` (new-req full data + cached-req diffs) — ported 1:1 from vLLM V1 on top of M1.3's KVCacheManager. This is what the EngineCore (M1.8) step loop calls. Behavioral (no model/CUDA); tests ported from upstream `tests/v1/core/test_scheduler.py`.

**Architecture:** Mirrored: `src/vllm/v1/core/sched/{scheduler,output,request_queue}.{h,cpp}` (← `vllm/v1/core/sched/`), `src/vllm/config/scheduler.{h,cpp}` (← `vllm/config/scheduler.py`, the T0 config subset). Scope T0: FCFS scheduling, chunked prefill, preemption, the `SchedulerOutput`/`NewRequestData`/`CachedRequestData` diff protocol, `update_from_output`. Defer priority scheduling, spec-decode, structured-output grammar, encoder, KV-connector, async-scheduling (T1/T2) behind marked stubs.

### Task 3: Scheduler.schedule() (the core algorithm)
Read `/home/mudler/_git/vllm/vllm/v1/core/sched/scheduler.py` `schedule()` + the helpers (`_try_schedule_encoder_inputs` defer; the running/waiting loops; preemption). Port `Scheduler(vllm_config-subset, kv_cache_config, ...)` holding `running` (list), `waiting` (RequestQueue), `kv_cache_manager` (M1.3), `scheduled_req_ids`, and `schedule() → SchedulerOutput`:
- Running loop: for each running request, `num_new_tokens = min(num_tokens_with_spec − num_computed, budget, long_prefill_threshold?)`; `allocate_slots(...)`; on None → preempt (FCFS tail pop, free KV, re-queue to waiting); build CachedRequestData diffs.
- Waiting loop (only if no preemptions this step): for each waiting request, `get_computed_blocks` (prefix hit), admit under budget + max_num_seqs, `allocate_slots`, build NewRequestData.
- Chunked prefill falls out of `min(num_new_tokens, token_budget)`. Defer: encoder budget, spec-decode tokens, structured-output, LoRA gating, KV-connector — marked stubs.
Unit tests ported from test_scheduler.py: single request prefill schedules under budget; chunked prefill splits a long prompt across steps; multiple requests share the budget; preemption when KV exhausted (allocate returns None → tail preempted + requeued); the SchedulerOutput shape (new vs cached).
