# M1.6 Paged Attention + CommonAttentionMetadata

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** The KV-cache-aware attention backend — `CommonAttentionMetadata` (the per-step struct carrying `query_start_loc`/`seq_lens`/`block_table`/`slot_mapping`/`max_*` to the kernels), the `AttentionBackend`/`AttentionMetadataBuilder` interface, a correctness-grade **paged attention** op (varlen prefill + paged decode reading K/V from the block-paged KV cache via `slot_mapping`/`block_table`), the **reshape_and_cache** op (write new K/V into the paged cache at `slot_mapping`), and the **GDN attention metadata** (prefill/decode segmentation for the hybrid gate models' mamba-state path) — ported 1:1 from vLLM V1. CPU + CUDA, validated against pinned-oracle goldens.

**Architecture:** Mirrored `src/vllm/v1/attention/{backend,backends/...}.{h,cpp}` (← `vllm/v1/attention/`) + the attention ops in `src/vt/`. This generalizes M0.9's dense causal attention to the **KV-cache-backed, batched, variable-length** case: instead of dense q/k/v, the op reads K/V from paged blocks indexed by `block_table` + writes new tokens via `slot_mapping` (`reshape_and_cache`). The GDN layers keep their mamba-state cache (M0.7 GDN ops) — M1.6 adds the metadata segmentation that routes a batched step into GDN's prefill (chunked scan) vs decode (recurrence) segments. Scope T0: the correctness-grade paged path the gate models need (bf16, GQA 16/2, the hybrid full-attn + GDN split); the perf kernels (FlashInfer-class, CUDA graphs) are M2.4.
