# M1.7 Sampler (V1 sampling pipeline)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** The V1 `Sampler` — the ordered sampling pipeline that turns the model's
final logits `[num_reqs, vocab_size]` into sampled token ids (+ optional
logprobs), driven by the per-slot `SamplingMetadata` built from M1.5's persistent
`InputBatch` — ported 1:1 from `vllm/v1/sample/`. This is what the model runner
(M1.8) calls after the forward. Scope T0: the correctness-grade sampling path the
gate models + the OpenAI-serving MVP need — **greedy (bit-exact argmax), temperature,
top-k, top-p, penalties (repetition/frequency/presence), min-tokens / logit-bias /
min-p logits processors, allowed-token-ids + bad-words masks, and logprobs
(top-k + ranks)**. CPU + CUDA, correctness-grade. Defer spec-decode rejection
sampling, thinking-budget, logprob_token_ids (generative-scoring), and the
FlashInfer/Triton fused samplers (M2.4) behind marked 1:1 stubs.

**Architecture:** Mirrored `src/vllm/v1/sample/{sampler,metadata,ops/...}.{h,cpp}`
(← `vllm/v1/sample/`) + the sampling primitives as `vt::` ops in `src/vt/`. The
Sampler consumes `SamplingMetadata` (the T0 field subset) and the `[num_reqs,vocab]`
logits produced by the M0.9 forward / the batched runner; it produces the existing
`SamplerOutput` (include/vllm/v1/engine/types.h — `sampled_token_ids` +
`logprobs_tensors`; the logprobs payload, currently an opaque flag, is fleshed out
here via a `LogprobsTensors` type ported from `vllm/v1/outputs.py`).

## Global Constraints

- 1:1 with pinned e24d1b24 — VERIFY the pinned file's CURRENT API before porting
  (proven-necessary habit). Same class/method/field names. Ported files carry
  `// Ported from: vllm/v1/sample/<file> @ e24d1b24`.
- **The pipeline ORDER is the correctness core** — match `Sampler.forward` +
  `Sampler.sample` EXACTLY (sampler.py docstring steps 1-9): raw logprobs snapshot
  BEFORE penalties/temperature → float32 → allowed-token-ids → bad-words →
  non-argmax-invariant procs (min_tokens, logit_bias) → penalties (rep/freq/pres) →
  sample{ greedy argmax snapshot; temperature (guard temp<eps→1.0 when not
  all_random); argmax-invariant procs (min_p); top_k_top_p; random exponential-noise
  gumbel-max; `where(temp<eps, greedy, random)` } → gather logprobs (top-k+sampled,
  ranks). `_SAMPLING_EPS = 1e-5`.
- **GREEDY IS THE PARITY GATE — bit-exact.** `greedy_sample = argmax(logits, dim=-1)`
  over the f32 logits; this must match upstream token-for-token (it already does at
  M0-exit for the assembled forward). Lowest-index tie-break (torch.argmax returns
  the first max) — match it.
- **RANDOM-SAMPLING RNG (key decision, record precisely):** upstream `random_sample`
  uses the **exponential-noise gumbel-max** trick — `q ~ Exponential(1)` per
  (req,vocab), pick `argmax(probs / q)` (equivalently `argmax(logits - log(q))`),
  with per-request `torch.Generator` seeds (Philox4x32) overriding the batch default.
  Port the EXACT ALGORITHM (exponential-noise gumbel-max, per-request seed override,
  the "reuse greedy tensor via where(temp<eps)" merge). **Bit-exact parity vs
  torch's Philox4x32 + its exponential transform is a DOCUMENTED DEFERRAL** (it needs
  a byte-identical Philox counter RNG + the exact `exponential_()` inverse-CDF) — at
  T0 we use a deterministic, reproducible-within-vllm.cpp seeded RNG that is
  distribution-correct and implements the same gumbel-max selection. Greedy stays
  bit-exact; random is validated by (a) algorithm/shape, (b) determinism under a
  fixed seed, (c) distribution (large-N frequency ≈ softmax probs). Record this as
  an accepted deviation in the ledger + a T1 carry (torch-Philox for exact random
  parity). Do NOT block M1.7 on bit-exact random parity.
- **Logits are the layer output** — validate sampling ops against composed reference
  math (argmax, log_softmax, sorted top-k/top-p masking, penalty formulas) the same
  way M1.6 validated paged attention: unit tests with hand-computed expecteds; where
  a pinned CPU path exists (apply_top_k_top_p_pytorch, apply_penalties), port its
  exact masking/formula and dump a small golden from the pinned oracle if cheap.
- Port upstream's OWN tests from `/home/mudler/_git/vllm/tests/v1/sample/` (cite —

- `apply_temperature(logits, temp, all_random)` — in-place div by temp, guard
  `temp<eps→1.0` when `!all_random` (per-row `where`).
- `greedy_argmax(logits) -> token_ids` — argmax dim=-1, **lowest-index tie-break**.
- `apply_top_k_top_p(logits, k, p)` — the sort-based pytorch path EXACTLY (sort
  ascending; top-k: threshold = sorted value at position `vocab-k`, mask `< thr`;
  top-p: softmax→cumsum, mask `cumsum <= 1-p`, force-keep last; scatter back). Plus
  the `apply_top_k_only` no-sort path for k-only. Per-row k/p (None = skip).
- `random_sample(probs/logits, generators)` — exponential-noise gumbel-max: draw
  `q~Exp(1)` per element (seeded per-request override), `argmax(probs/q)`. Seeded,
  deterministic, distribution-correct (see Global Constraints RNG decision).
- `compute_logprobs(logits) = log_softmax(logits, f32)`.
