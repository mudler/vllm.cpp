# M3.5 C API + library packaging (libvllm)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** The llama.cpp-style **library-first packaging** the MVP mandates — a stable
C ABI (`include/vllm.h`) over the C++ engine, built as a shared + static `libvllm`,
plus an example CLI, so downstreams (LocalAI via purego/cgo, any C/FFI consumer)
can load and drive vllm.cpp without the C++ headers. Scope T0: load a model + run
completion (greedy + sampled, streaming callback + blocking), the minimal handle-
based C API. This is MVP gate #3 (library-first packaging).

**Architecture:** `include/vllm.h` — a pure-C header (no C++ types leak; opaque
handles + POD structs + `extern "C"`). `src/capi/vllm_c.cpp` — the C ABI impl
wrapping the M1.8 `LLMEngine` + M3.1 serving pieces. CMake produces `libvllm.so` +
`libvllm.a`. An `examples/cli/main.cpp` (a `main`-style `complete`/`chat` CLI) and
a dlopen/FFI smoke test prove consumption. No new engine logic — this is a thin,
ABI-stable wrapper.

## Global Constraints

- **Pure C ABI**: `include/vllm.h` uses only C types (opaque `typedef struct
  vllm_* vllm_*;` handles, POD param structs, `const char*`, primitive ints/floats,
  function-pointer callbacks). `extern "C"` guard. NO C++ in the header. Stable,
  versioned (a `VLLM_ABI_VERSION` + `vllm_version()`), documented ownership (who
  frees what) for every returned pointer.
- Errors via return codes (`vllm_status`) + a thread-local `vllm_last_error()`
  string — never throw across the ABI (wrap every C++ call in try/catch → status).
- Mirror the SHAPE of llama.cpp's C API ergonomics where sensible (handle-based
  init → context → generate → free), but the semantics are our LLMEngine.
- Warnings-as-errors; TDD; push authorized (CPU CI). The dlopen smoke test proves
  the ABI is loadable + drives a generation.

### Task 1: The C ABI surface (`include/vllm.h`) + core impl (init/free/complete)
Design + port `include/vllm.h` (pure C) + `src/capi/vllm_c.cpp`:
- Handles: `vllm_engine` (owns the LLMEngine + its Scheduler/runner/KV/processors
  stack), `vllm_model_params` / `vllm_sampling_params` (POD structs mirroring the
  T0 fields: model path, block_size, num_blocks, max_model_len; temperature, top_p,
  top_k, max_tokens, seed, stop strings, min_p, penalties).
- `vllm_engine_load(const vllm_model_params*, vllm_engine** out) -> vllm_status`:
  build the full engine stack (config/tokenizer/weights → LLMEngine) — reuse the
  examples/server load path (factor the model-load into a shared helper if not
  already). `vllm_engine_free(vllm_engine*)`.
- `vllm_complete(vllm_engine*, const char* prompt, const vllm_sampling_params*,
  char** out_text, ...) -> vllm_status` (blocking; caller frees out_text via
  `vllm_string_free`). Return the completion text + finish reason + token counts.
- `vllm_status` enum + `vllm_last_error() -> const char*` (thread-local); every
  entry point try/catch → status, never throws across the boundary.
- `vllm_version()` + `VLLM_ABI_VERSION`.
Unit tests (C++-side, calling the C API): load a synthetic model (reuse the M1.8/
M3.1 test harness weights via a test-only loader entry, OR a `vllm_engine_load`
variant that accepts in-memory synthetic weights for the test), run vllm_complete
greedy → deterministic text; error path (bad model path → status + last_error, no
throw); string ownership (free without leak — ASan clean).

### Task 2: Streaming callback API + sampled generation
Add `vllm_complete_stream(vllm_engine*, const char* prompt, const
vllm_sampling_params*, vllm_token_callback cb, void* user_data) -> vllm_status`
where `typedef bool (*vllm_token_callback)(const char* delta_text, bool finished,
void* user_data)` (return false to stop early). Drives the engine's DELTA output →
the callback per token/delta; matches the OpenAI streaming semantics but as an FFI
callback. Sampled path (temperature>0 + seed) works. Unit tests: streaming
callback deltas concatenate to the blocking result; early-stop (cb returns false)
terminates; a seeded sampled run is deterministic.

### Task 3: CMake shared/static lib + example CLI + dlopen smoke
- CMake: produce `libvllm.so` (SHARED) + `libvllm.a` (STATIC) exporting only the C
  ABI symbols (a version script / visibility=hidden + `VLLM_API` export macro so
  the C++ internals aren't exported). Install rules (`vllm.h` + the libs).
- `examples/cli/main.cpp`: a `complete`/`chat` CLI using the C API (dlopen or
  link) — the llama.cpp-style `./vllm-cli --model ... --prompt ...`.
- **dlopen smoke test**: a test (or script) that `dlopen("libvllm.so")` +
  `dlsym` the ABI functions + loads a tiny synthetic model + runs vllm_complete —
  proving a purego/cgo-style consumer works without the C++ headers. (This is the
  DoD: "LocalAI-style consumption via dlopen/purego".) Gate behind a CMake option
  if dlopen/shared-lib isn't available in the core CI.
Unit/smoke tests + the CLI build under warnings-as-errors.
